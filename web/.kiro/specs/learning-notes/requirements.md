# 学习笔记页面需求文档

## 项目概述
为个人技术展示网站添加学习笔记/知识库功能，用于记录和展示技术学习路线图和详细学习内容。

## 用户故事

### US-1: 导航菜单项
**作为** 网站访问者  
**我想要** 在导航栏看到"学习笔记"或"知识库"菜单项  
**以便于** 快速访问学习内容

**验收标准:**
- [ ] 导航栏在"博客"后面增加新菜单项
- [ ] 菜单项名称待确认："学习笔记" 或 "知识库"
- [ ] 点击菜单项跳转到学习笔记页面
- [ ] 当前页面时显示激活状态（与其他菜单项一致的样式）

### US-2: 技术栈导航
**作为** 学习者  
**我想要** 看到所有技术栈的列表（Linux, Docker, PHP, Redis, ELK）  
**以便于** 选择我想查看的技术内容

**验收标准:**
- [ ] 页面左侧显示技术栈导航列表
- [ ] 包含5个技术栈：Linux, Docker, PHP, Redis, ELK
- [ ] 当前选中的技术栈有高亮显示
- [ ] 显示当前学习进度或状态（可选功能，待确认）
- [ ] Docker 为默认选中项（当前学习重点）

### US-3: 章节结构展示
**作为** 学习者  
**我想要** 看到课程式的章节结构  
**以便于** 系统化地学习技术内容

**验收标准:**
- [ ] 右侧主内容区显示章节列表
- [ ] 每个章节可以展开/收起
- [ ] 章节标题清晰可见
- [ ] 默认状态待确认（全部展开 or 全部收起）

### US-4: Docker 学习路线图
**作为** Docker 学习者  
**我想要** 看到完整的 Docker 学习路线  
**以便于** 从基础到实战系统学习

**验收标准:**
- [ ] 包含7个主要章节：
  1. 基础入门 - Docker 概念、安装配置
  2. 镜像管理 - 镜像操作、Dockerfile
  3. 容器操作 - 容器生命周期管理
  4. 网络配置 - Docker 网络模式
  5. 数据管理 - 数据卷、持久化
  6. Docker Compose - 多容器编排
  7. 实战项目 - Ubuntu Linux 部署实践
- [ ] 每个章节包含详细的学习内容
- [ ] 内容详细程度待确认（概览 vs 详细教程）

### US-5: 学习内容展示
**作为** 学习者  
**我想要** 查看每个章节的详细内容  
**以便于** 深入学习具体知识点

**验收标准:**
- [ ] 点击章节可展开查看详细内容
- [ ] 内容包含文字说明
- [ ] 代码示例需要语法高亮（待确认是否需要）
- [ ] 内容排版清晰易读
- [ ] 支持 Markdown 格式（可选）

## 技术实现要点

### 路由配置
- 路径: `/notes` 或 `/learning`
- 组件: `src/modules/learning-notes/views/LearningNotes.vue`
- 路由模式: 懒加载

### 数据结构
```javascript
{
  techStack: [
    {
      id: 'docker',
      name: 'Docker',
      status: 'learning', // learning | completed | planned
      progress: 60, // 可选
      chapters: [
        {
          id: 'docker-basics',
          title: '基础入门',
          sections: [
            {
              title: 'Docker 是什么',
              content: '...',
              code: '...' // 可选
            }
          ]
        }
      ]
    }
  ]
}
```

### 样式要求
- 保持与现有页面一致的设计风格
- 使用玻璃态效果（glass-panel）
- 响应式设计，支持移动端
- 平滑的展开/收起动画

### 依赖库（可选）
- 代码高亮: highlight.js 或 Prism.js
- Markdown 渲染: markdown-it（如需要）
